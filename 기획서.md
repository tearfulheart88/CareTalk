# 돌봄톡 (CareTalk) — 카카오톡 네이티브 독거노인 AI 돌봄 에이전트

> **AGENTIC PLAYER 10 공모전 출품작**
> 버전: v2.1 (본선 MVP 완료) | 최종 갱신: 2026-06-23

---

## 1. 프로젝트 한 줄 요약

**"카톡 친구 추가만 하면 시작됩니다 — 앱 설치 0%, 학습 0%. 말 한마디로 안부를 전하고, AI가 24시간 살핍니다."**

독거노인이 카카오톡 채널 친구 추가만으로 매일 안부 확인, 건강 기록, 위험 감지, 추억 회상 대화, 가족 연결을 하나의 대화창에서 이용하는 MCP 기반 AI 돌봄 서비스입니다.

---

## 2. 문제 정의

### 2.1 사회적 배경

| 지표 | 수치 | 출처 |
|------|------|------|
| 독거노인 가구 수 | 228.8만 가구 (2024년) | 통계청 |
| 65세+ 카카오톡 사용률 | 65.7% | 카카오 데이터 |
| 독거노인 응급안전확인 서비스 대상 | 약 53만명 | 보건복지부 |
| 응급안전확인 누락률 | 약 76% (신청 대상 대비) | 국회예산정책처 |

### 2.2 핵심 문제

1. **기존 돌봄 서비스는 앱 설치·IoT 장비 설치가 필요** → 65세+ 사용자가 포기
2. **인력 기반 안부 확인은 비용·확장성 한계** → 지자체 예산 부족
3. **자녀는 부모님 상태를 실시간으로 알 수 없음** → 정보 단절
4. **독거노인의 정서적 고립** → 우울증, 자살률 증가

### 2.3 돌봄톡의 해결책

| 문제 | 기존 방식 | 돌봄톡 |
|------|----------|--------|
| 접근성 | 앱 설치·회원가입 | 카톡 친구 추가만 |
| 안부 확인 | 인력 전화 (주 1~2회) | AI 매일 자동 + 응답 감정 분석 |
| 위험 감지 | 버튼 누름 / 움직임 센서 | 대화 텍스트에서 위험 키워드 실시간 감지 |
| 건강 관리 | 종이 기록 / 별도 앱 | 카톡 대화로 자연어 건강 기록 |
| 정서 지원 | 복지사 월 1회 방문 | AI 추억 회상 대화 (24시간) |
| 가족 연결 | 전화·문자 | 알림톡 + Widget 대시보드 |

## 3. 타겟 사용자

### 1차 사용자: 독거노인 (65세+)
- **필요**: 매일 안부 확인, 건강 관리, 누군가와 대화하기
- **특징**: 스마트폰 사용은 카톡이 거의 전부. 앱 설치·학습 어려움
- **UX 원칙**: 큰 글씨, 음성 입력, 긴급 호출 버튼, 따뜻하고 느린 말투

### 2차 사용자: 가족 (자녀)
- **필요**: 부모님 건강·정서 상태 실시간 확인
- **특징**: 타지에서 근무, 자주 방문 어려움
- **UX 원칙**: 알림톡 요약, Widget 대시보드, 위험 시 즉시 알림

### 3차 사용자: 복지사·지자체
- **필요**: 담당 구역 노인 상태 일괄 파악, 자원 배분 우선순위
- **특징**: 담당 인원 부족, 수기 관리에 의존
- **UX 원칙**: 주간 리포트, 위험 등급별 정렬

## 4. 핵심 기능 (MCP Tools)

### 4.1 예선 MVP (4개 Tool)

#### Tool 1: `daily_checkin` — 일일 안부 확인
- **목적**: 매일 아침 안부 메시지 발송 + 응답 감정 분석
- **Actions**:
  - `initiate`: 안부 메시지 발송 (시간대별 맞춤 인사 + 날씨)
  - `analyze`: 응답 텍스트 감정 분석 (positive/neutral/negative) + 건강 키워드 추출 + 위험 키워드 감지
  - `no_response`: 무응답 확인 → 가족 알림톡 트리거
- **입력**: `user_id`, `action`, `message`(선택), `nickname`(선택)
- **출력**: `status`, `sentiment`, `health_keywords`, `follow_up_action`
- **예시**:
  ```
  사용자: "오늘 기분이 좀 꿀꿀해요... 무릎도 아프고"
  → sentiment: negative
  → health_keywords: ["무릎", "아픔"]
  → follow_up_action: "주의 — 건강 키워드 감지. health_log 기록 권장"
  ```

#### Tool 2: `emergency_detect` — 위험 신호 실시간 감지
- **목적**: 대화 텍스트에서 위험 키워드 실시간 감지 + 긴급 레벨 판정
- **Actions**:
  - `detect`: 메시지 위험 판정 (3단계: GREEN/YELLOW/RED)
  - `silence`: 무응답 경보 (설정 시간 미응답 시)
- **위험 키워드 분류**:
  - RED (즉시 119): 쓰러졌어, 의식, 숨 못 쉬, 피, 골절
  - YELLOW (가족 알림): 아파, 어지러, 안 좋아, 힘들어, 외로워
  - GREEN: 정상 대화
- **출력**: `risk_level`, `detected_keywords`, `recommended_action`, `notify_targets`

#### Tool 3: `family_report` — 가족용 돌봄 리포트
- **목적**: 자녀에게 주간/일일 안부·건강·감정 리포트 전송
- **Actions**:
  - `weekly`: 7일간 안부 응답률, 감정 추이, 건강 키워드, 위험 이력 종합
  - `daily`: 당일 안부 응답 + 감정 + 건강 기록 요약
- **출력**: 카카오 BasicCard JSON + `summary_text` + `alert_items`
- **알림톡 연동**: `summary_text`를 알림톡 본문으로 발송

#### Tool 4: `daily_care_widget` (Widget A) — 노인용 "오늘의 돌봄"
- **목적**: 노인용 매일 아침 카톡 메시지 카드
- **구성**: SimpleText (안부 인사 + 날씨 + 연속 응답 일수) + quickReplies (건강체크, 전화, 식단)
- **출력**: 카카오 스킬 응답 v2.0

### 4.2 본선 확장 (3개 Tool)

#### Tool 5: `health_log` — 건강 데이터 기록·추세 분석 ⭐ 본선

**목적**: 혈압·혈당·체중·체온·맥박을 자연어로 기록하고 추세를 분석

**4가지 Actions**:

| Action | 기능 | 입력 | 출력 |
|--------|------|------|------|
| `log` | 수치 직접 기록 | `data_type`, `value` | `status`, `normal_range`, `advice` |
| `query` | 기간별 조회 | `data_type`(선택), `days` | 기록 리스트 |
| `analyze` | 추세 분석 | `data_type`, `days` | `patterns`, `overall_status`, `trend_alert` |
| `parse` | 자연어 파싱 기록 | `message` | 파싱 결과 + 기록 결과 |

**지원 data_type**: `systolic`(수축기), `diastolic`(이완기), `blood_sugar`(혈당), `weight`(체중), `temperature`(체온), `heart_rate`(맥박)

**정상/주의/위험 3단계 판정 기준**:

| data_type | 정상 | 주의 | 위험 |
|-----------|------|------|------|
| 수축기 혈압 | <120 | 120-139 | ≥140 |
| 이완기 혈압 | <80 | 80-89 | ≥90 |
| 혈당 | 70-140 | 141-180 / 60-69 | >180 / <60 |
| 체온 | 36-37.2 | 37.3-37.9 | ≥38 / <35 |

**자연어 파싱 예시**:
```
입력: "오늘 혈압 135/85요"
→ 파싱: 수축기 135, 이완기 85
→ 기록: systolic=135 (주의), diastolic=85 (주의)
→ advice: "수축기/이완기 모두 주의 구간입니다. 식습관 관리를 권장합니다."

입력: "아까 혈당 재봤는데 220이네"
→ 파싱: 혈당 220
→ 기록: blood_sugar=220 (위험)
→ trend_alert: "🚨 위험 구간입니다. 즉시 의사 상담을 권장합니다."
```

**추세 분석**: 최근 N일간 데이터로 급격한 변화(일간 변화폭 >20%) 감지 시 trend_alert 발송

#### Tool 6: `reminiscence_chat` — 추억 회상 정서 지원 대화 ⭐ 본선

**목적**: 감정 상태에 맞춘 추억 회상 주제로 정서 지원 대화

**2가지 Actions**:

| Action | 기능 | 입력 | 출력 |
|--------|------|------|------|
| `chat` | 회상 대화 응답 | `message`, `sentiment`, `nickname` | `response_text`, `suggested_topic`, `suggested_media` |
| `suggest_topic` | 맞춤 주제 추천 | `sentiment`, `nickname` | `suggested_topic`, `topic_prompt` |

**감정별 맞춤 회상 주제 (9종)**:

| 감정 | 추천 주제 | 예시 |
|------|----------|------|
| positive | 가장 행복했던 날, 좋아하던 음식, 좋아하던 노래 | "손주 처음 봤을 때 어떤 느낌이었어요?" |
| neutral | 고향 풍경, 학교 시절, 첫 직장 | "고향에서 가장 좋아하던 장소는요?" |
| negative | 따뜻했던 기억, 자랑스러웠던 일, 가족과의 추억 | "가장 따뜻했던 기억 한 가지를 들려주세요" |

**키워드 기반 후속 질문**: 사용자 메시지에서 키워드(학교, 직장, 고향, 음식, 계절 등) 추출 → 관련 후속 질문 자동 생성

**GPT 연동**: OPENAI_API_KEY 있으면 GPT-4o-mini로 따뜻한 회상 대화 생성, 없으면 템플릿 폴백

**대화 기록**: 모든 회상 대화는 checkin_responses 테이블에 `[회상]` prefix로 저장 → family_report에서 정서 추이 데이터로 활용

#### Tool 7: `family_report_widget` (Widget B) — 가족용 "주간 돌봄 리포트" ⭐ 본선

**목적**: 자녀가 카톡에서 부모님의 주간 상태를 한눈에 파악

**구성**:
1. **BasicCard**: 주간 요약
   - 안부 응답률 (예: 85.7%, 7일 중 6일 응답)
   - 감정 추이 (긍정 4회 / 보통 2회 / 나쁨 1회)
   - 주요 건강 키워드 (예: 혈압 3회, 수면 2회)
   - 위험 감지 이력 (RED: 0건, YELLOW: 1건)
2. **ListCard**: 일별 상태 목록 (최대 7일)
   - 날짜 + 감정 이모지 + 메시지 요약
3. **버튼**: 전화하기 (phone action), 상세 리포트 보기 (message action)

## 5. 기술 아키텍처

### 5.1 전체 구조

```
[카카오톡 채널]
    ↓ 사용자 메시지
[MCP Client (카카오 스킬 서버)]
    ↓ JSON-RPC 2.0
[돌봄톡 MCP Server (Python http.server)]
    ↓ tools/call
[7개 Tool 구현체]
    ↓                ↓                ↓
[SQLite DB]    [GPT-4o-mini]    [알림톡 API]
(schema.py)    (gpt_service)    (alimtalk)
```

### 5.2 MCP 서버

- **프로토콜**: JSON-RPC 2.0 (MCP 표준)
- **엔드포인트**: `/mcp` (POST)
- **메서드**: `initialize`, `tools/list`, `tools/call`, `notifications/initialized`
- **의존성**: 순수 Python stdlib `http.server` (외부 패키지 0)
- **포트**: 9000번대 (8000번대 충돌 방지)

### 5.3 데이터베이스 (SQLite)

**단일 진실원천**: `db/schema.py`의 `ensure_schema(db_path)` 하나에서 모든 테이블 관리

| 테이블 | 용도 |
|--------|------|
| `users` | 사용자 정보 (user_id, nickname, phone, family_contact) |
| `checkins` | 일일 안부 발송 기록 |
| `checkin_responses` | 안부 응답 + 감정 분석 결과 (회상 대화 포함) |
| `emergency_logs` | 위험 감지 이력 |
| `silence_alerts` | 무응답 경보 |
| `family_reports` | 가족 리포트 발송 이력 |
| `alerts` | 알림 큐 |
| `health_logs` | 건강 데이터 기록 (혈압/혈당/체중/체온/맥박) |

### 5.4 외부 서비스 연동

| 서비스 | 파일 | Mock 폴백 | 비고 |
|--------|------|-----------|------|
| GPT-4o-mini | `services/gpt_service.py` | ✅ 규칙기반 | 감정 분석, 회상 대화, 위험 컨텍스트 |
| 카카오 로그인 | `services/kakao_auth.py` | — | OAuth 2.0 (REST API) |
| 카카오 알림톡 | `services/alimtalk.py` | ✅ 콘솔 출력 | 비즈메시지 API |

> **Mock 모드**: OPENAI_API_KEY 없이도 전 기능 동작. API 키가 있으면 GPT 호출, 없으면 규칙기반 폴백.

## 6. 사용 시나리오 (End-to-End)

### 시나리오 1: 일상 안부 확인

```
[오전 8:00] 돌봄톡: "🌞 좋은 아침이에요, 순자님! 오늘도 화이팅이에요~"
            [Widget A: 안부 인사 + 날씨 + 퀵릴라이]

[오전 8:30] 순자: "오늘 기분이 좋아요! 산책도 했어요"

[돌봄톡 내부]
  → daily_checkin(analyze): sentiment=positive, health_keywords=["산책"]
  → 긍정 응답 → 가족 리포트에 "긍정" 누적

[오전 9:00] 순자: "아까 혈압 재봤는데 135/85요"

[돌봄톡 내부]
  → health_log(parse): systolic=135(주의), diastolic=85(주의)
  → "수축기/이완기 모두 주의 구간입니다. 식습관 관리를 권장해요~"

[오후 3:00] 순자: "옛날 고향 생각이 나네..."

[돌봄톡 내부]
  → reminiscence_chat(chat, sentiment=neutral):
    "고향 생각이 나시는군요. 고향에서 가장 좋아하셨던 장소는 어디였어요?"
  → suggested_topic: "고향 풍경"
```

### 시나리오 2: 위험 감지 → 가족 알림

```
[오후 2:00] 순자: "아이고, 방금 욕실에서 넘어졌어... 너무 아파"

[돌봄톡 내부]
  → emergency_detect(detect):
    risk_level: RED
    detected_keywords: ["넘어졌어", "아파"]
    recommended_action: "🚨 RED 경보: 즉시 119 신고 + 가족 긴급 알림톡"
    notify_targets: ["family", "119", "welfare_officer"]

[즉시]
  → 알림톡 발송 (자녀): "🚨 긴급: 순자님께서 위험 상황입니다..."
  → 자녀 알림톡에 "전화하기" 버튼
```

### 시나리오 3: 주간 가족 리포트

```
[일요일 오전] 자녀 카톡에 알림톡 도착:

  [Widget B: 주간 돌봄 리포트]
  BasicCard:
    "👵 순자님 주간 돌봄 리포트
     기간: 2026-06-16 ~ 2026-06-23
     안부 응답률: 85.7% (7일 중 6일)
     감정 추이: 😊 긍정 4 / 😐 보통 1 / 😔 나쁨 1
     주요 건강 키워드: 혈압(3회), 수면(2회)
     ⚠️ 주의 일: 1일 (06-21 무릎 통증 호소)"

  ListCard:
    06-23 😊 좋음 — "오늘 기분이 좋아요! 산책도 했어요"
    06-22 😐 보통 — "그저 그래요..."
    06-21 😔 나쁨 — "무릎이 너무 아파요..."
    ...

  [📞 전화하기] [📋 상세 리포트]
```

## 7. 차별화 포인트

### 7.1 기술 차별화

| 항목 | 기존 돌봄 서비스 | 돌봄톡 |
|------|-----------------|--------|
| 진입 장벽 | 앱 설치 / IoT 설치 | 카톡 친구 추가 (0%) |
| 안부 확인 | 인력 전화 (주 1~2회) | AI 매일 자동 + 감정 분석 |
| 위험 감지 | 버튼 / 센서 하드웨어 | 대화 텍스트 실시간 NLP |
| 건강 기록 | 종이 / 별도 앱 | 카톡 자연어 ("혈압 135/85요") |
| 정서 지원 | 복지사 월 1회 | AI 추억 회상 (24시간) |
| 가족 연결 | 전화 / 문자 | 알림톡 + Widget 대시보드 |

### 7.2 MCP 표준 준수

- JSON-RPC 2.0 표준 프로토콜
- `tools/list` → `tools/call` 표준 패턴
- PlayMCP 등록 가능 (카카오클라우드 배포 후)
- 외부 AI 에이전트가 돌봄톡 Tool을 호출 가능

### 7.3 비용 효율성

| 항목 | 기존 (IoT 기반) | 돌봄톡 (SW 기반) |
|------|----------------|-----------------|
| 초기 설치비 | 30~50만원/가구 | 0원 (카톡 친구추가) |
| 월 유지비 | 1~3만원/가구 | 5천~1만원/가구 (GPT API) |
| 확장성 | 하드웨어 조달 필요 | 서버만 확장 |
