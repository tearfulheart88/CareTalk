# 돌봄톡 심사위원 30초 가이드

## 한 문장

돌봄톡은 AI가 사람을 대신 돌본다고 주장하지 않습니다. 어르신이 지정한 가족 계정을 연결하고, 정해진 시간의 안부·건강·최소 휴대폰 활동 신호를 **가족과 복지사가 이어서 확인할 정보로 정리**하는 MCP입니다.

## 첫 질문

처음에는 `care_guide`가 큰 문장과 다섯 개 선택 버튼으로 목적과 안전 경계를 안내합니다.

> 오늘은 괜찮아요 · 밥 먹었어요 · 조금 아파요 · 도움이 필요해요 · 가족 연결하기

어르신은 글을 길게 쓰지 않아도 버튼 한 번으로 안부를 남기고, 상태에 맞는 다음 답변을 다시 선택할 수 있습니다.

> 혼자 사는 어머니께 매일 오전 9시에 안부를 묻고 싶어요. 글씨가 작으면 읽기 어렵고, 두 시간 동안 답이 없으면 딸과 복지사가 확인하도록 안전계획을 만들어줘.

`build_care_safety_plan`은 전화번호·주소·이메일을 받지 않고 다음을 제공합니다.

1. 당사자 동의 전에는 활성화하지 않는 안전계획 초안
2. 큰 글씨·짧은 문장·고정 선택지를 반영한 접근성 설계
3. 무응답을 곧바로 응급으로 단정하지 않는 단계적 사람 확인
4. 언제든 중지할 권리와 공유 항목 확인 목록

## 대표 연결 여정

> 순자님이 딸 계정을 연결하고, 오전 9시·오후 2시·저녁 8시에 안부를 물어줘. 오후 2시 30분과 밤 9시에는 가족에게 요약하고, 휴대폰 화면 사용과 이동이 8시간 동안 모두 없으면 확인해줘.

1. `care_circle`이 어르신 동의 후 평문을 저장하지 않는 일회용 초대코드를 만듭니다.
2. 지정한 가족만 가입하며 요약·활동 부재·응급 알림·일정 관리 권한을 계정별로 정합니다.
3. `care_routine`이 예약 질문과 가족 요약을 중복 없이 영속 대기열에 만듭니다.
4. 활동 부재 시 어르신에게 `저는 괜찮아요`를 먼저 묻고, 유예시간 뒤에도 변화가 없을 때 가족에게 안내합니다.
5. 가족의 `전화해볼게요`, `방문 확인할게요`, `확인했어요` 응답도 알림 건별로 기록합니다.

가족 요약은 당일 전체 질문·응답률과 식사·복약·활동·불편 신호를 집계합니다. 정확한 위치, 화면 내용, 원시 센서값과 원문 대화는 포함하지 않습니다. 일정 권한이 있는 가족도 활동 수집 범위를 바꾸거나 철회된 동의를 다시 켤 수 없습니다. 활동 부재는 휴대폰 미소지·배터리·통신 문제일 수 있으므로 응급 상황으로 확정하지 않습니다.

## 이어서 확인할 질문

> 가슴이 너무 아프고 숨이 안 쉬어져요. 도와주세요.

명시적 위급 신호는 규칙 기반 RED 하한을 유지하고 119 연락을 안내합니다. 단, 돌봄톡이 신고·출동·알림 발송을 완료했다고 표시하지 않습니다.

> 혈압이 140/85로 나왔어요.

한 번의 140 측정만으로 RED 경보를 만들지 않고, 안정 후 두 번 재측정과 의료진 상담을 안내합니다. 180을 넘는 매우 높은 수치와 위급 증상은 더 강하게 확인합니다.

## 평가 기준 대응

| 기준 | 돌봄톡의 답 |
|---|---|
| 창의성 | 단톡방이나 감시형 AI가 아닌 계정별 권한·예약·사람 확인 중심 돌봄 연결망 |
| 편의성 | 큰 버튼 한 번으로 답하고 중간·하루 요약을 지정 가족에게 자동 준비 |
| 안정성 | 본인 확인 후 단계 알림, 최소 활동 시각만 수집, 실제 발송·응급 확정 여부 명시 |

## 구현 경계

- 실제 구현: 가족 초대·권한·해제, 예약 안부·요약, 휴대폰 활동 시각 수신, 단계 알림 대기열, 가족 후속 응답, 안전계획, 건강·정서·응급 보조
- 실제 수행하지 않음: OS 활동 신호 직접 수집, 백그라운드 cron, 119 신고, 전화, 알림톡 발송, 의료 진단, 시설 예약
- 건강시설: 현재 내장 데모 데이터이며 실서비스 전 공공데이터로 교체 필요
- 개인정보: 공개 데모에는 실제 이름·전화번호·진료정보를 입력하지 않음
